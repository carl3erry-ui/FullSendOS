## Purpose

Define client memory architecture as a cross-engagement strategic capability.

## Memory Domains

- Brand
- Voice
- Financials
- Employees
- Customers
- Markets
- Competitors
- Past engagements and reports
- Logos and style guide
- Historical KPIs
- Research evidence
- Investor materials

## Behavioral Rules

- New engagements should inherit approved client context.
- Stable client facts should not require re-entry each engagement.
- Memory updates should be traceable to validated work products.
- Unsupported assumptions should not be promoted to stable memory facts.

## Integration with Departments

Departments consume relevant client memory as context and produce updated candidate memory artifacts for review.

## Quality and Safety

- Distinguish verified facts from assumptions.
- Preserve source traceability where available.
- Avoid exposing sensitive material in user-facing logs or diagnostics.
